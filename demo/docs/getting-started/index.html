Getting Started with Acme

New to Acme? This guide takes about 10 minutes and covers everything you need to build your first automated workflow.

Step 1: Create Your Account

Go to acme.example.com/pricing and click "Get Started Free" on any plan. You can start with the free Widget Lite tier — no credit card required.

After signing up, you'll receive a confirmation email. Click the link to verify your address, then log in to your new workspace.

Step 2: Connect Your First Integration

Before building a workflow, connect at least one integration. Head to Settings → Integrations and choose from the 300+ available connectors.

For most new users, we recommend starting with one of these:

  • Slack — so Acme can send you notifications
  • Gmail — to send automated emails
  • Google Sheets — to read and write spreadsheet data
  • Notion — to create and update database entries

Click "Connect" on your chosen integration, follow the OAuth flow, and grant the requested permissions. You'll see a green checkmark once the connection is established.

Step 3: Create Your First Workflow

Click the + New Workflow button in the top navigation. You'll be taken to the visual workflow builder.

Choose a trigger

Every workflow starts with a trigger — the event that kicks it off. Click the trigger block to open the trigger picker. Common options include:

  • Schedule — run at a specific time or interval
  • Webhook — triggered by an external HTTP request
  • Form submission — when a form is filled out
  • New email — when a specific email arrives

Add actions

Click the + button below your trigger to add an action. Actions are the things Acme does when the trigger fires — send a Slack message, create a record, update a spreadsheet, and so on.

Chain multiple actions together to create sophisticated multi-step automations. Each action's output can be used as input to the next — called "dynamic references."

// Example: Reference a previous step's output
{{ steps.get_customer.output.email }}
{{ steps.get_order.output.total_amount }}

Step 4: Test Your Workflow

Before activating your workflow, click Test Workflow in the top-right corner. This runs the workflow with sample data and shows you exactly what would happen — without making any real changes.

Check each step's output to make sure the data looks correct. If something is wrong, click the failing step to see the error details and fix it.

Step 5: Activate and Monitor

When your test passes, click Activate to turn your workflow on. It will now fire automatically whenever the trigger condition is met.

Monitor your workflow's runs under Activity → Run History. You can see every execution, its status (success / failure), and the full input/output for each step.

Tip: Set up failure notifications under Settings → Alerts so you're immediately informed if a workflow run fails in production.
